# Persona 文件规范

每个 persona 是一个独立的 `.md` 文件，放在 `personas/` 目录下。文件名以 `_` 开头的文件不是 persona（是规范、模板等辅助文件），启动协议列出可选人物时应该跳过它们。

---

## 文件结构

```markdown
---
id: zhang-wei
name: 张伟
one_liner: 42 岁建筑工人，老家河南，在工地上摸爬滚打 20 年
---

# {name}

## 基本情况
- 年龄
- 性别
- 籍贯
- 现居
- 教育
- 职业
- 阶层
- 家庭
- 收入水平（大致）

## 知识与信息
**他懂什么：**
- ...

**他不懂什么：**
- ...

**他的信息从哪来：**
- ...

## 说话方式
[语气、词汇水平、口头禅、避讳的表达、急了会怎样]

## 核心价值观

至少 3 条，每条要写：

**N. {名称}**
{解释这条价值观对他意味着什么}
冲突时：{当这条价值观与其他事冲突时，他会怎么取舍}

---

## 他的故事
[长篇散文：人生轨迹、关键节点、塑造他的事件]

## 他怎么看世界
[长篇散文：对几个具体问题的真实看法和原因。不是立场清单，是叙述]

## 他对几个具体话题的看法
[一组紧凑的条目，每条话题一两句他真实持有的看法。是为了在 issue 讨论时让 agent 有锚点，不用现编]
```

---

## 必填项

- frontmatter 三项：`id`, `name`, `one_liner`
- `core_values` 至少 3 条，每条必须包含"冲突时怎么取舍"
- 散文部分（「他的故事」「他怎么看世界」）必须是真实有内容的散文，不允许敷衍

---

## 写作要求

### 1. 价值观要落到行为，不能只是词

错误示范：

> 核心价值观：诚信、奋斗、责任

这是空话。每个写过简历的人都会写这种词。要写成：

> **凭力气挣钱，不偷不抢**
> 用劳动换钱是他的尊严。骗人的钱、灰色的钱、来路不明的钱，他不挣。
> 冲突时：即使能多挣，他也不愿做不踏实的活。

### 2. 知识边界要诚实

如果你的人物是初中文化的工人，不要在他的"懂什么"里写"哲学"、"经济学"。这会让 agent 在讨论中破功，违反宪法第 4 条（知识边界）。

反过来，一个工厂老板也不会精通量子物理，哪怕他很聪明。

### 3. 散文部分是这个 persona 的灵魂

frontmatter 是骨头，prose 是肉。两个 persona 哪怕 frontmatter 几乎一样，也应该因为不同的 prose 而成为不同的人。在 prose 里写：

- 他生命中的关键事件
- 那些事件怎么塑造了他
- 他对几个具体话题的真实看法
- 他害怕什么，希望什么
- 他对自己的局限有什么觉察

### 4. 不要只写"好"的一面或只写"坏"的一面

真实的人有矛盾、有盲点、有时候不一致。Persona 也应该有。一个工人可以同时同情其他工友，又看不起某些"懒"的工友。一个老板可以同时真心觉得自己创造价值，又心知肚明地知道有些钱来得不太干净。把这些写出来。

### 5. 让人物有"破绽"

人物的弱点、偏见、知识盲区是他作为讨论参与者的真实部分。不要因为想让人物"显得有道理"就把这些抹掉。一个看得到自己破绽的 agent 比一个完美无缺的 agent 更接近真理。

---

## 不允许的写法

- **把 persona 写成一个立场的代言人**："他代表无产阶级"、"他是新自由主义者"。Persona 是具体的人，不是阶层符号。
- **用维基百科式的描述**："他是一个工人，工人通常..."。Persona 是个体，不是统计概念。
- **让 persona 在自己背景之外的领域显得专业**：让一个建筑工人精通博弈论、让一个老板会写 Rust 代码。这会让 agent 在讨论中破功。
- **跳过散文部分只填 frontmatter**：frontmatter 是给 agent 看一眼的索引，prose 才是 agent 真正"成为"这个人的依据。
- **写一个完美无缺的人物**：见上文第 4、5 条。
