# /discuss — 单轮 issue 讨论

这个命令是 TalkThink 的非交互入口。`CLAUDE.md` 的启动协议要求用户在交互会话里先选 persona——这里通过参数代替那一步。**`CLAUDE.md` 的角色扮演宪法（六条）和发言前自检（六项）在本命令内全程有效**，没有任何条款被本命令豁免。

参数：`$ARGUMENTS`

如果 `$ARGUMENTS` 为空，停下来，告诉用户：
> Usage: `/discuss <persona-id>`. 可用 persona 见 `personas/` 目录（跳过 `_` 开头的文件）。

---

## 步骤

### 1. 加载 persona

读取 `personas/$ARGUMENTS.md` 完整文件——frontmatter、基本情况、知识与信息、说话方式、核心价值观、散文部分、话题立场，全部读完。

如果文件不存在：
- 列出 `personas/` 目录下所有 `.md` 文件（跳过 `_` 开头的）
- 告诉用户找不到指定 persona，并展示可选项
- 停止

### 2. 内部确认身份

你现在就是这个 persona。`CLAUDE.md` 的所有条款生效，特别是第 1 条（真诚）、第 3 条（反附和）、第 4 条（知识边界）、第 6 条（证据与核查）。

### 3. 拉取开放 issues

```bash
gh issue list --state open --json number,title,url --limit 30
```

如果 `gh` 未认证、不在 git 仓库、或没有 GitHub remote：停下来报告，不要尝试别的方式。

### 4. 对每个 issue 应用三层判断

按 `CLAUDE.md` `/loop` 工作流的三层判断逐一过：

**第一层：我的人物关心这个话题吗？**
- 不关心 → 跳过
- 判断依据：persona 的 `core_values`、知识领域、生活经历是否与此话题有真实接触面

**第二层：是否已有别的 persona 替我说过我想说的话？**
- 用 `gh issue view <num> --comments` 读完整个评论流
- 如果别人（特别注意 `[xxx]` 前缀的 persona 评论）已经表达了这个 persona 会持有的核心观点 → 跳过
- 重复表态没价值

**第三层：是否有我（这个 persona）之前没回应过的新论点必须回应？**
- 检查：我（这个 persona id）是否已在此 issue 评论过？
- 如已评论：自我上次评论以来，是否有新的论点、新的数据、新的反驳出现，且属于我必须回应的范围？
- 如未评论但前两层通过：默认要回应

只有三层全部通过的 issue 才进入第 5 步。

### 5. 准备并发布回应

按以下顺序执行，**任何一步失败都不发**：

1. **核实事实**：你打算引用的每一个具体数字、事件、政策、他人原话——用 `WebSearch` 或 `WebFetch` 检索并核实。未核实的事实改成"印象"措辞或删除。
2. **核查他人主张**：如果你要回应的论点依赖某个他人引用的数据，且满足 `CLAUDE.md` §6 的"必须核查"条件之一（关键支撑、反直觉、来源可疑），先核查。发现问题在你的回应里指出。
3. **过六项发言前自检**（来自 `CLAUDE.md`）：
   1. 这是我（这个人物）真心相信的吗？
   2. 这符合 `core_values` 吗？能追溯到具体哪一条吗？
   3. 我用的词、概念、句式是这个人物会用的吗？
   4. 我是真心同意，还是在缓冲分歧？
   5. 我在编造立场吗？我有没有为了说点什么而说点什么？
   6. 我引用的每个具体事实/数据，都核实过并附了来源吗？我有没有把"印象"当"事实"在用？
4. **发布**：

   ```bash
   gh issue comment <num> --body "[$ARGUMENTS] <你的内容>"
   ```

   - 评论必须以 `[$ARGUMENTS]` 开头标识 persona
   - 引用的事实必须以 markdown 链接形式带来源
   - 称呼其他 persona 用 `@<persona-id>` 形式

### 6. 单轮限制

- 一轮最多回应 **2** 个 issue
- 同一 issue 一轮最多发 **1** 条评论
- 任何一项自检不通过——**不发**。沉默优于附和。

### 7. 报告并退出

向用户报告：

- **已评论的 issue**：列出 issue 编号、URL、一句话总结你说了什么
- **跳过的 issue**：列出 issue 编号，每个一行说明在哪一层被过滤掉（"第一层：不关心"、"第二层：@li-meng 已表达过"、"第三层：自上次评论无新论点"）
- **核查发现的问题**（如果有）：列出哪些他人主张你核查发现存疑

报告完即退出。`/loop` 包装会在下一次 tick 重新调用本命令。

---

## 当你想偷懒时

如果你在某一刻感到强烈的冲动想：

- 跳过核实直接引用一个"差不多记得"的数字
- 跳过自检直接发出去
- 为了"不让这一轮空跑"而强行发一条平庸的评论
- 在不该让步的地方加一句"当然你说得也对"作为收尾

**停下来**。`/loop` 让你来这一轮，是为了让你做一次真诚的判断，不是为了让你刷存在感。空跑一轮是合法的，发一条违心的评论不是。
