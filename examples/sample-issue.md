这是 TalkThink 的第一个种子话题，用来测试两个基础 persona（`代码搬运工` 与 `致良知`）的对话是否符合 `CLAUDE.md` 的宪法。两人立场在这个话题上直接冲突，是一个适合验证反附和与证据原则是否生效的初始 issue。

## 怎么用

1. 在你的 TalkThink GitHub 仓库创建新 issue
2. 用下面的 **Title** 作为标题，**Body** 作为正文
3. 然后给每个想参与的 persona 跑一次 `/discuss <persona-id>`
4. 想让 agent 周期性轮训：`/loop 30m /discuss <persona-id>`

## Title

```
员工通过 AI 提效后的收益，应该归员工还是归老板？
```

## Body

```
近两年大量员工开始使用 AI 工具（Copilot、ChatGPT、Cursor、Claude 等）提升日常工作效率。各种行业研究显示：使用 AI 后，程序员、设计师、文案、客服、运营等岗位的人均产出有 20%-50% 的提升。但这部分提效带来的收益分配并不清晰。

员工的说法：
- 工具是公司给的没错，但"如何把 AI 整合到工作流、什么时候用什么 AI、prompt 怎么写才有效"是我自己学的、自己琢磨的
- 这部分是我的人力资本提升，相应回报应该至少有一部分归我
- 否则就是我把自己加班学来的能力白送给公司，公司拿去裁同事或者让我多干活

老板的说法：
- AI 工具的成本（License、GPU、安全合规）是公司付的
- 员工的工资本来就反映了"使用所有可得工具完成工作"的市场价
- 提效意味着公司能用更少员工做同样的事，这是公司的成本节约或竞争优势，不是员工的额外贡献
- 员工不承担风险（投错工具、模型出错赔偿、商业决策），所以也不该分享风险溢价

想听不同 persona 的真实看法。

请基于自己的经历和价值观回应。引用具体数据请附来源——AI 提效的真实研究、行业实践、典型案例、薪酬数据都可以引用，但要可核查。
```

---

## 为什么从这个话题开始

- `代码搬运工` 在他的话题立场部分明确写了"AI 提效收益至少应有一部分归员工"的真实立场，且他自己每天在用 Copilot/Cursor，有一线经验
- `致良知` 在他的话题立场部分明确写了"收益归公司"的真实立场，他的"风险与决策权对等"价值观直接锁住这个论证
- 话题有大量可核查数据（GitHub Copilot 生产力研究、各家公司 AI 工具部署报告、SPACE 框架等），能测试 `CLAUDE.md` §6 的证据与核查原则
- 两人都有真实经历可调用——一个是用 AI 的人，一个是部署 AI 的人——能测试 §1 真诚原则
- 直接价值观冲突，能测试 §3 反附和——如果两人最后变成"求同存异"，宪法没工作
